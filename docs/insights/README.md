# Insights Feature Ideas

Ideas for surfacing meaningful data to users and admins. Each file covers one category.

| File | Audience | Summary |
|------|----------|---------|
| [personal_progress.md](personal_progress.md) | User | Progression snapshot, consistency breakdown, what to work on next |
| [community_intelligence.md](community_intelligence.md) | User | Crowd-sourced difficulty ratings, leash position consensus, trick popularity |
| [style_profile.md](style_profile.md) | User | Position tendencies, transition preferences, exploration gaps |
| [admin_platform_health.md](admin_platform_health.md) | Admin | Moderation queue, content quality gaps, user engagement stats |

## Priority order (suggested)

1. **Unlocked tricks not yet attempted** (`personal_progress.md`) — immediately actionable, builds on data already tracked
2. **Community difficulty divergence** (`community_intelligence.md`) — surfaces value from `difficulty_vote` data already being collected
3. **Leash position consensus** (`community_intelligence.md`) — same, uses `leash_position` already tracked
4. **Tricks missing metadata** (`admin_platform_health.md`) — low effort, high content quality payoff
